    详细使用说明
    
            
    
          
    

          
    
            
    1. 原理是什么
    
            
    Plex 入库时主要认“路径 + 文件名 + 目录结构”，并不立刻校验真实视频内容。本工具在本地生成与真实媒体同名、同目录结构的 1KB 假文件，让 Plex 先快速扫库；扫完后再把 Plex 容器挂载切回真实文件路径即可播放。
    
            
      
              
    • 源目录：真实视频所在位置（可来自网盘/远程挂载）
      • 
              
    • 目标目录：假文件输出位置（请选本地硬盘）
      • 
              
    • 默认只生成视频占位文件（mp4/mkv/ts/m2ts 等），图片、字幕、nfo 等会跳过
      • 
            
    
          
    

          
    
            
    2. 面板怎么用
    
            
      
              
    1. 左侧选源目录，点“添加为源目录”
      2. 
              
    2. 右侧选目标目录（本地硬盘），点“添加为目标目录”
      3. 
              
    3. 在“待生成任务”里确认配对（第 1 源对应第 1 目标）
      4. 
              
    4. 选增量/全量与速度，点“开始生成假文件”
      5. 
              
    5. 在右侧日志查看进度；生成完成后把假文件目录挂给 Plex 扫库
      6. 
            
    
          
    

          
    
            
    3. 生成后如何给 Plex 替换路径
    
            
    关键点：Plex 容器内的挂载路径必须前后一致，只切换宿主机真实目录。
    
            
    
              
    
                A
                
    
                  阶段一：假文件入库
                  
    把“目标目录下生成的假文件结构”映射进 Plex，例如：
    
                  
    # docker-compose 示例（路径按你的实际环境改）
# 工具生成结果类似：
# /本地/115/CloudDrive/影音库/...

# Plex 假文件阶段
- /本地/115/CloudDrive:/media/library
    
                
    
              
    
              
    
                B
                
    
                  阶段二：切换真实文件
                  
    入库完成后，保持 Plex 容器内路径不变，只改宿主机真实源：
    
                  
    # Plex 真实文件阶段（容器内仍是 /media/library）
- /真实网盘/CloudDrive:/media/library
    
                
    
              
    
              
    
                C
                
    
                  检查
                  
      
                    
    • 容器内路径前后一致（例如都是 /media/library）
      • 
                    
    • 目录层级、文件名与假文件阶段一致
      • 
                    
    • 切换后在 Plex 刷新/分析媒体库即可播放
      • 
                  
    
                
    
              
    
            
    
          
    

          
    
            
    4. 假文件入库的优点 / 缺点
    
            
    
              
    
                
    优点
    
                
      
                  
    • 扫库快，几百 TB 也能较快入库
      • 
                  
    • 减少短时间对网盘/远程带宽的冲击
      • 
                  
    • 路径切换后可无缝继续使用已入库条目
      • 
                
    
              
    
              
    
                
    缺点 / 注意
    
                
      
                  
    • 入库阶段不能正常播放（内容是 1KB 假文件）
      • 
                  
    • 媒体信息、海报、时长等可能不完整或不准确
      • 
                  
    • 必须严格保证切换前后路径一致，否则会变成“新条目”
      • 
                  
    • 目标目录请选本地硬盘，选网盘会很慢且不稳定
      • 
                
    
              
    
            
    
          
    

          
    
            
    5. 模式、速度与常见问题
    
            
      
              
    • 增量：目标已有假文件会跳过；全量：覆盖重写视频假文件
      • 
              
    • 源目录扫描速度：网盘建议 20–100 个/秒（防封控）；本地源可更高；0 表示不限速
      • 
              
    • 看不到目录：先确认 docker-compose 已映射 volumes，再点“刷新挂载点”
      • 
              
    • 生成很慢：目标改到本地硬盘，并适当降低速度
      •
